REST Lab

Use the REST Lab to test your API, without having to write a one-off test program:

  1. Select End Point type: your end points are your Base Tables, Views, Procedures (these are all created automatically), plus your Custom Resources. Choose a type, then choose an End Point.

  2. URL: this is the URL you can use in REST calls, e.g., a mobile app. You can add URL parameters for filter, sort and so forth (see docs, and the Quick Ref on the left Navigation Bar - de-select Short Menu).

  3. Send: Fill in the request as appropriate, and click the type of call (GET, PUT etc) to run.

The Logs (Left Navigation Bar) will show you the SQLs and Rules resulting from your request.
