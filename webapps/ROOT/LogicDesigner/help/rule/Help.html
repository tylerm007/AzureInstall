Reactive Logic


What
  1. Logic enforces database integrity on RESTful update requests - Business Rules for derivations, validations, integration, and events.

  2. Logic is specified using Reactive Programming, plus a familiar server side JavaScript event model, so there are no limits on what you can do.

  3. Reactive Logic is conceptually similar to spreadsheet cell formulas:
    • You bind expressions to database columns
    • On RESTful updates, (Post, Put, Delete), the system watches for changes to dependent data, and reacts with automatic Change Propagation.
    • Rules apply to all updates - both tables, and resources.


  4. Reactive logic has key differences to conventional procedural code:
    • Invocation is automatic - the engine (not your code) calls the logic - it never forgets
    • Automatic Change Propagation to dependent data - including related tables
    • Automated SQL and transaction bracketing for Change Propagation to related tables

Why
Reactive eliminates tedious boilerplate code for Change Detection and Propagation, Persistence and Transaction Management.

As a result, a small handful of reactive logic is equivalent to many hundreds of lines of conventional code.
