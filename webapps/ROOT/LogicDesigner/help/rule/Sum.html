Sum


Sums declares a Parent attribute to be maintained as the sum of a designated child role's summed attribute, with an optional qualification to filter child objects. Parent reference names are derived from Foreign Key Relationships.

For example, to derive the balance as the sum of the unshipped orders:

Derive balance as
       sum(purchaseorderList.amount_total
       where null === ShippedDate)

Sums are

  1. Meaningful reductions in logic, due to automatic Change Detection and Propagation, including SQL and Transaction Management.

  2. Efficient - parent data is adjusted via a 1 row update - the system does not issue a select sum() SQL command
