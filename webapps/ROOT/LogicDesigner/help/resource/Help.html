Resources

When you connect, the system creates Default End-points for each table, view and stored procedures. You can control access using Security.

You can also define Custom End-points, e.g. for custom apps, or for data integration Web Services. You can explore the pre-created samples.

Observe:

  1. Custom Resources provide a tree-like document model

  2. You can choose and rename the objects and attributes, so your API is truly custom, not merely a flat reflection of your relational tables

  3. Sub Resources can combine data from multiple data sources using Resource Types (see sample Resources in the Logic Demo project):
    • Sql
    • MongoDB
    • Other RESTful services


  4. Logic and Security apply “underneath” the RESTful API, so are enforced for Live Browser, or any RESTful API call, whether through default or Custom End-points

  5. Your API includes Enterprise-class pattern automation::
    • Pagination
    • Optimistic Locking
    • Generated Primary Key Propagation to related child data, etc.


  6. No codegen or deployment, so your resources are operational as soon as you click save

  7. Test them in the REST Lab
